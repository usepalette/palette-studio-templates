# Playbooks

How we think about things. The team's shared brain.

A playbook captures the philosophy, frameworks, and approach for a domain. It's not step-by-step instructions — it's the thinking behind the doing.

## Format

Each playbook is a markdown file named by domain:

```
playbooks/
├── sales.md     # how we think about selling
└── content.md   # how we think about content
```

A playbook typically covers:

- **Philosophy** — What we believe about this domain
- **Frameworks** — Mental models and decision-making tools
- **Approach** — How we generally tackle work in this area
- **Preferences** — Team-specific choices and style (grows over time)

## When to write one

Write a playbook when you notice the team keeps making the same kind of decision, and you want to codify how to think about it. Don't write them in advance — write them when real work demands it.
